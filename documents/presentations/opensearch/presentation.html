Was ist OpenSearch?

  • Entwickelt von Elasticsearch
  • Basiert auf einem Fork alter Versionen von Elasticsearch und Kibana
  • Erweiterbar und flexibel
  • Zum suchen, analysieren und visualisieren von Daten

Vorteile

  • Opensource und gratis
  • Dokumentation und Tutorials stehen zur Verfügung
  • Aktive community
  • Foren und Github zur Hilfe

Use Cases generell

  • Volltextsuche

    • Inhalte auf Websites, in Dokumenten, Datenbanken oder Anwendungen untersuchen.

  • Log- und Ereignisanalyse

    • Opensearch wird häufig zur Analyse von Log Daten eingesetzt und hilft somit DevOps Systemprobleme zu erkennen.

  • Data analytics

    • Opensearch kann eingesetzt werden um durch die Analyse von großen Mengen an Daten eine Übersicht über Geschäftsmetriken und Trends bereitzustellen.

Use Case in unserem Project - Eisbär!

  • Passende Produkte aus der Datenbank liefern.

    • Opensearch hilft uns dabei aus der riesigen Datenbank ein passendes Resultat für spezifische Kundeanfragen zu liefern.

    Wir nutzen also hauptsächlich die Volltextanalyse.

Getting Started

  1. OpenSearch installieren
  2. OpenSearch starten (commandline)
  3. Produktdaten für OpenSearch in ausgewähltem Format vorbereiten (zb.: JSON)
  4. OpenSearch API verwenden um die Daten zu indexieren.
  5. OpenSearch in das System integrieren und durch HTTP - Requests im Backend Suchanfragen an OpenSearch stellen.

Conclusion

OpenSearch ist ein starkes Tool, das wir verwenden können um für bestimmte Beschreibungen eines Produkts das richtige Ergebnis aus der Datenbank zu finden. Ebenfalls kann es genutzt werden um Log Daten zu analysieren, damit die DevOps einen besseren Überblick über mögliche Systemfehler haben und diese schneller gelöst werden können.
